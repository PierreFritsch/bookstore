---

The pull request description should contain:

- a description of the issue: 
- how to reproduce the issue from scratch, including how to clone the repo: When I ... and I ..., then <what actually happens>. I would expect <what should happen instead>.
- for backend bugs, add a failing test and explain how to run it
- for frontend bugs, add a failing test (Playwright) and explain how to run it
- for frontend bugs, consider adding screenshots to the issue description
